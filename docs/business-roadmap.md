# 业务规划：观察资产 + 个人持仓 + 分级买卖建议

本文档定义 Personal Invest 下一阶段的业务主线。系统不只是股票研究工具，也不只是基金分析工具，而是个人投资研究、持仓管理、组合复盘和分级买卖建议系统。

## Summary

系统需要提供买卖建议，但第一阶段采用“分级建议”，不是黑盒交易指令。建议由规则生成，AI 负责解释，所有建议必须带依据、风险、置信度和适用边界。

核心业务主线：

- 观察资产：我正在研究什么。
- 个人持仓：我真实持有什么。
- 组合复盘：风险和暴露在哪里。
- 分级建议：现在更适合观察、持有、买入关注、减仓关注，还是卖出关注。

## 业务模型

### 观察资产

观察资产替代“自选股”，表示用户正在研究、跟踪或准备复盘的投资对象。

第一阶段支持：

- `STOCK`：股票，公司研究对象。
- `ETF`：场内 ETF / LOF，交易型基金资产。
- `FUND`：场外公募基金，净值型资产。

观察资产不等于已经买入。它回答“是否值得继续研究”，不是“是否已经持仓”。

### 个人持仓

个人持仓表示用户真实持有的资产。第一阶段只支持一个默认组合，不拆券商账户、基金账户或现金账户。

股票持仓记录：

- 代码
- 名称
- 数量
- 成本价
- 当前价
- 市值
- 盈亏
- 仓位
- 行业

ETF 持仓记录：

- 代码
- 名称
- 份额
- 成本价
- 当前价
- 市值
- 盈亏
- 跟踪方向

场外基金持仓记录：

- 基金代码
- 基金名称
- 份额
- 成本净值
- 最新净值
- 市值
- 盈亏
- 基金类型
- 风险等级

### 组合复盘

组合复盘回答“我的风险集中在哪里”，不是只展示持仓表格。

组合层需要解释：

- 股票、ETF、场外基金分别占多少。
- 单一资产是否过度集中。
- 行业、主题、市场方向是否过度集中。
- 哪些持仓出现回撤、趋势转弱、风险上升。
- 哪些观察资产值得进入下一步研究。

## 分级买卖建议

第一阶段建议等级使用固定枚举：

| 建议等级 | 含义 |
|---|---|
| `继续观察` | 数据不足或信号不强，保留观察但不提高优先级 |
| `买入关注` | 条件改善，进入买入候选或重点研究 |
| `持有` | 持仓条件未恶化，继续跟踪 |
| `减仓关注` | 风险上升或集中度偏高，需要考虑降低暴露 |
| `卖出关注` | 核心条件明显恶化，需要优先复核退出逻辑 |

每条建议必须包含：

- 建议等级
- 触发原因
- 关键指标
- 风险说明
- 置信度
- 数据日期

建议由规则生成，AI 只解释规则结果。AI 不直接凭空生成建议。

### 今日建议表达

今日建议不能只展示“买入关注”“减仓关注”“卖出关注”等标签，必须给出具象化解释，让用户知道为什么今天要看它。

每条今日建议至少包含：

- 一句话结论：用自然语言说明今天为什么进入该建议等级。
- 建议等级：继续观察、买入关注、持有、减仓关注、卖出关注。
- 触发来源：趋势、估值、行业、资金、回撤、仓位、风险事件等。
- 关键证据：列出 2-4 个最重要指标。
- 风险提醒：说明该建议可能失效的原因。
- 复核动作：告诉用户下一步应该看什么，而不是只给标签。

示例表达：

| 建议等级 | 具象化表达 |
|---|---|
| `买入关注` | 趋势连续改善，所属行业强度进入前列，但估值已经不低，适合加入候选并等待回调或确认信号。 |
| `持有` | 持仓趋势仍然健康，当前盈亏和仓位没有触发风险线，继续跟踪行业强弱和成交变化。 |
| `减仓关注` | 单一资产仓位偏高，同时近期波动放大，建议优先复核是否降低组合暴露。 |
| `卖出关注` | 趋势转弱并触发风险事件，且持仓回撤扩大，需要优先复核退出条件。 |
| `继续观察` | 数据不足或信号不够一致，暂时不提升优先级，继续观察价格、行业和风险变化。 |

## 资产建议口径

股票建议依据：

- 趋势
- 估值
- 行业强弱
- 公司质量
- 成交资金
- 风险事件
- 持仓盈亏

ETF 建议依据：

- 跟踪方向强弱
- 趋势
- 波动
- 最大回撤
- 流动性
- 市场环境适配

场外基金建议依据：

- 净值表现
- 近 1/3/6/12 月收益
- 最大回撤
- 波动
- 基金类型
- 风险等级
- 持有体验

持仓资产额外考虑：

- 仓位占比
- 盈亏
- 回撤
- 集中度

观察资产额外考虑：

- 是否值得加入候选
- 是否继续跟踪
- 是否降低关注

## 市场与行业全景分析

系统需要同时分析市场环境和行业结构，不能只展示少数热门行业。

市场分析回答：

- 当前市场状态是积极、谨慎还是防守。
- 市场宽度是否改善，强势资产数量是否增加。
- 成交量能是否支持当前趋势。
- 指数趋势和行业结构是否一致。
- 当前更适合提高关注、保持持有，还是降低暴露。

行业分析回答：

- 热门行业：哪些行业当前强度高、资金活跃、趋势延续。
- 冷门行业：哪些行业持续弱势、成交低迷、风险未解除。
- 轮动行业：哪些行业从弱转强或从强转弱。
- 防守行业：哪些行业在市场转弱时相对抗跌。
- 过热行业：哪些行业涨幅过快、波动放大、回撤风险升高。
- 主题映射：热门行业对应哪些股票、ETF 或基金观察对象。

行业分析需要覆盖“强、中、弱”全谱系，而不是只列前几名。Dashboard 和日报展示摘要，行业页展示完整排名、分组和解释。

## 产品方向

Dashboard 首屏展示：

- 市场环境
- 行业全景摘要
- 组合风险
- 今日关注资产
- 今日建议

今日建议展示时，必须直接写出具象化原因。例如：

- 买入关注：趋势改善 + 行业升温 + 风险未触发。
- 减仓关注：仓位偏高 + 波动放大 + 行业转弱。
- 卖出关注：趋势破位 + 风险事件 + 回撤扩大。

观察池展示：

- 资产类型
- 当前建议等级
- 关注理由
- 最新风险
- 下一步观察点

个股分析页只输出股票建议。

基金分析页输出 ETF 和场外基金建议，但口径分开：

- ETF 使用价格型 ETF 口径。
- 场外基金使用净值型基金口径。

持仓页升级为组合风险与操作建议页。

日报包含：

- 市场判断
- 持仓风险
- 资产建议
- 明日复盘清单

AI 分析用于解释“为什么给这个建议”，并展示建议依据。

## 验收标准

业务文档和后续实现必须能回答：

- 系统是否给买卖建议？给，但采用分级建议。
- 建议由谁生成？规则生成建议，AI 解释建议。
- 是否直接自动交易？不做自动交易。
- 股票、ETF、基金是否共用建议规则？不共用，只共用建议等级框架。
- 观察池和持仓有什么区别？观察池是研究对象，持仓是真实资产。
- 持仓页核心是什么？组合暴露、盈亏、集中度、风险和建议。
- 每条建议是否可追溯？必须可追溯到指标、规则、数据日期和风险说明。

## Assumptions

- 第一阶段只支持一个默认个人组合。
- 建议等级先用固定枚举，不做个性化策略模板。
- 股票和 ETF 可共享价格行情管线。
- 场外基金使用独立净值管线。
- 后续实现前，先保证业务规划、信息架构、产品 backlog 和 task board 口径一致。
