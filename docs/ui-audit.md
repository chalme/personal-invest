# Web UI 审计报告

## Summary

- Audit Date: 2026-06-21
- Target: `https://invest.chalme.indevs.in/`
- Mode: 线上只读审计
- Viewport: Desktop, 1440px wide
- Pages: Dashboard、个股分析、持仓、观察池、复盘、日报、设置
- Browser Console: 未发现 error / warn

结论：当前 UI 不是功能缺失，而是信息层级过满。系统已经具备“结论优先、风险优先、低可信数据提示、复盘闭环”的产品方向，但视觉组织仍像功能型 MVP：左侧导航过长，页面标题重复，卡片层级相近，灰色行动块过重，表格和表单抢占决策信息。

下一步应按 `UX-006 -> UX-007 -> UX-008 -> UX-009` 推进，不建议直接逐页微调。

## Overall Findings

### P0: 主导航过于平铺

左侧导航目前有 14 个入口：

```text
Dashboard / 市场趋势 / 行业强弱 / 个股分析 / 基金分析 / 观察池 / 持仓 / 复盘 / 策略配置 / 回测 / 策略信号 / 日报 / AI 分析 / 设置
```

这会让系统看起来像功能清单，而不是投资工作台。建议在 `UX-006` 中收敛为：

```text
今日 / 观察 / 持仓 / 复盘 / 报告 / 设置
```

市场、行业、信号、策略、AI、回测作为二级入口或页面内模块。

### P0: 首屏层级太多

多数页面都同时出现：

- 顶部通用标题
- 页面标题
- 大卡片
- 多个摘要卡
- 表格
- 表单
- 状态标签

用户能看到很多信息，但不容易判断“现在最该看什么”。建议统一首屏结构：

```text
页面结论
关键风险
可选动作
证据摘要
明细
```

### P1: 组件语言不稳定

目前存在多种状态表达：

- `HIGH` / `MEDIUM`
- `SINGLE_POSITION_HIGH`
- `FINANCIAL_DEBT_HIGH`
- `样本数据`
- `混合数据`
- `数据缺失`
- `否 / 低置信`

这些信息本身有价值，但用户侧不应大量展示原始规则代码。建议 `UX-007` 统一：

- 用户看到中文状态和简短解释。
- 原始 rule code 放入次级元信息或 tooltip。
- 数据可信度统一为 `REAL` / `ESTIMATED` / `SAMPLE` / `MISSING` / `MIXED` 加中文解释。

### P1: 深色行动块过重

Dashboard、持仓、日报等页面有大量深灰色块。它们看起来像强行动作或禁用状态，但实际很多只是信息入口。这会造成视觉噪声。

建议：

- 只给真正主动作使用强背景。
- 信息摘要使用轻背景、细边框或表格式排布。
- 风险状态用语义色和优先级，而不是统一深灰。

## Page Findings

### Dashboard

现状：

- 已经具备“今日投资判断”和“今日重要事项”，方向正确。
- 首屏同时出现市场判断、样本数据提示、数据可信度、重要事项、最近决策、指标卡。
- Full page 中约有 28 个 card-like 区块，信息密度高但优先级不清。

问题：

- 通用 H1 “个人投资研究系统”占据页面最高层级，Dashboard 自身标题不够明确。
- 样本数据提示和可信度提示连续出现，视觉重量接近重要事项。
- 重要事项列表直接展示 raw code，例如 `SINGLE_POSITION_HIGH`。
- “先处理 5 条风险事件”容易把系统变成待办工具，与低摩擦复盘方向不完全一致。

建议进入 `UX-008`：

- 首屏保留 5 个问题：数据是否新鲜、市场如何、组合是否有风险、哪些事项需要看、明天关注什么。
- 将数据可信度压缩为一条状态栏，详情跳转设置页。
- 重要事项只展示 Top 3，剩余进入复盘页。
- 将 raw code 转成用户语言，例如“单一持仓过高”。

### 个股分析

现状：

- 已经有“研究结论”，并按判断、证据、风险、下一步组织，方向很好。
- 图表、评分雷达、财报快照、结构化结论和评分列表都已覆盖。

问题：

- 默认选中的资产是沪深300ETF，但页面是“个股公司分析”。这会破坏 STOCK / ETF 的分析边界。
- 研究结论内部多层嵌套卡片，信息之间视觉权重过于接近。
- “数据缺失”重复出现，但没有形成统一可信度组件。
- 底部表格可用，但当前选中行和上方结论的关系还可以更强。

建议进入 `UX-009`：

- 股票页默认只展示 `STOCK`，ETF 应导向基金/ETF 入口。
- 研究结论顶部使用统一的资产结论组件。
- 财报缺失状态统一为数据可信度组件。
- 表格保留，但作为明细，不压过结论。

### 持仓

现状：

- 已经从纯表格升级为组合决策结论。
- 能看到最大风险、优先复核、集中度、资产类型暴露和复盘入口。

问题：

- “新增 / 更新持仓”表单位于页面中段，视觉重量过高，会抢走组合风险主线。
- 复盘入口用了大块深色卡片，像主操作区，但信息含义并不一致。
- 持仓明细表中建议、复核动作、风险、操作挤在一起，扫描成本偏高。

建议进入 `UX-009`：

- 默认页面以组合风险和持仓复核为主。
- 新增/更新持仓收进次级区域或折叠面板。
- 持仓表按“标的、仓位、盈亏、建议、风险、动作”重新分组。
- 删除按钮降低视觉重量，避免和主要复核动作竞争。

### 观察池

现状：

- “观察池研究状态”已经比纯列表更清楚。
- 能看到正在研究、数据待补齐、可能失效、资产分布。

问题：

- 新增/更新观察标的表单占据较大面积，压过研究状态。
- 表格中的研究状态和下一步观察文案有价值，但视觉层级较平。
- 观察池缺少“今天新变化”或“关注已失效”的更强摘要。

建议进入 `UX-009`：

- 将新增表单折叠或移动到次级操作。
- 首页区域优先展示“继续观察、待补数据、降低关注”三组。
- 表格保留为完整资产清单。

### 复盘

现状：

- 复盘页是当前产品闭环最完整的页面。
- 有重要事项、状态筛选、确认、延后、解决、记录决策。

问题：

- 事项数量多时页面很长，重复按钮很多。
- 每条事项展示 raw code、规则来源、过期日期、多个动作按钮，视觉密度过高。
- 高优先级和中优先级的视觉差异还不够清楚。

建议进入 `UX-009`：

- 将事项卡分成三层：标题与优先级、原因摘要、展开后的规则详情。
- 默认只显示 OPEN / 高优先级事项。
- 状态筛选改为更紧凑的 segmented control。
- “记录决策”保留，但其他动作可降低视觉重量。

### 日报

现状：

- 已经有“今日投资简报”，比纯 Markdown 阅读器好很多。
- 能看到最重要变化、是否复核、明日先看、数据边界。

问题：

- 简报和 Markdown 原文视觉差异不足，右侧原文区域过淡且过长。
- 市场/行业和资产分区像两列正文，不够像投资简报。
- 风险事件和明日关注仍混在长文本里。

建议进入 `UX-009`：

- 将简报固定为首屏主内容。
- Markdown 原文作为归档折叠或次级阅读。
- 风险、资产、行业、明日关注使用结构化摘要块。

### 设置

现状：

- 数据可信度总览最成熟，表格信息完整。
- 风控、数据源、AI、页面偏好集中在一个页面，符合设置页定位。

问题：

- 数据可信度表信息量大，作为设置页首屏合理，但扫描压力较高。
- 顶部操作按钮样式偏原生，和其他页面按钮语言不一致。
- `否 / 低置信` 多次出现，建议用统一 Badge。

建议进入 `UX-007`：

- 设置页可作为数据可信度组件规范的样板。
- 统一顶部按钮样式。
- 统一可信度和参与建议的展示方式。

## Recommended Next Steps

1. `UX-006`: 先改导航，不要先逐页美化。
2. `UX-007`: 再统一组件语言，尤其是可信度、风险、按钮和表格。
3. `UX-008`: 然后重做 Dashboard 信息架构。
4. `UX-009`: 最后逐页打磨股票、持仓、观察池、复盘、日报、设置。

## Acceptance Notes

- 不建议做营销页式 redesign。
- 不建议做 hero 大图或复杂动效。
- 不建议引入新前端框架。
- 不建议在数据可信度未统一前做强视觉包装。
- UI 优化重点是减少认知负担，而不是降低信息密度。
