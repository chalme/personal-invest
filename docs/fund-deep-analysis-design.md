# 场外基金深度分析设计

本文档定义场外基金深度分析的第一版设计。该能力只适用于 `FUND`，不包含 ETF / LOF 深度分析。

## 目标

场外基金分析要从“净值、收益、回撤、波动”的 MVP 口径，升级为“产品、经理、公司、策略、风险收益、基准比较、同类比较和持有体验”。

第一版要回答：

- 这个基金为什么值得继续观察或降低关注？
- 基金经理和基金公司是否稳定可信？
- 收益是否来自稳定超额，还是只是跟随市场上涨？
- 回撤、波动和回撤修复是否符合用户承受能力？
- 风格是否稳定，是否和现有持仓形成重复暴露？
- 基金异常是否需要进入重要事项池？

## 适用边界

- 只处理 `asset_type = FUND` 的场外基金。
- ETF / LOF 已有 `ETF_PRICE` 基础分析，但深度分析由 `P2-005` 单独规划。
- 场外基金不使用股票财报、公司估值或公司质量评分。
- AI 只解释基金深度数据、评分依据和规则结果，不直接根据基金资料自由生成买卖建议。
- 基金经理和基金公司数据缺失时，不能生成确定性管理能力结论。

## 数据模型规划

建议新增以下数据层：

- `fund_profile`：基金画像，覆盖基金类型、风险等级、基金经理、基金公司、成立时间、规模、基准、费率和申赎规则。
- `fund_manager_profile`：基金经理画像，覆盖任职时间、管理规模、历史产品和稳定性。
- `fund_company_profile`：基金公司画像，覆盖公司规模、产品线、管理资产和风险记录。
- `fund_risk_return_snapshot`：风险收益快照，覆盖阶段收益、最大回撤、波动、夏普、卡玛、下行波动和回撤修复时间。
- `fund_benchmark_snapshot`：基准比较快照，覆盖相对基准收益、跟踪差异和超额稳定性。
- `fund_peer_rank_snapshot`：同类比较快照，覆盖同类排名、同类分位和排名稳定性。
- `fund_holding_exposure_snapshot`：持仓暴露快照，覆盖股票、债券、货币、商品、海外比例，以及行业、主题、地区和风格暴露。

所有快照都应至少包含：

- `symbol`
- `name`
- `nav_date` 或 `snapshot_date`
- `fund_type`
- `source`
- `source_mode`
- `data_version`
- `created_at`

## 分析口径

基金画像：

- 基金类型和风险等级。
- 基金经理、基金公司、成立时间和规模。
- 基准、费率、申赎规则。
- 是否适合当前用户的持仓周期和风险偏好。

风险收益：

- 近 1 / 3 / 6 / 12 月收益。
- 最大回撤和回撤修复时间。
- 波动、下行波动、夏普、卡玛。
- 收益与回撤是否匹配。

基准和同类比较：

- 是否跑赢基准。
- 是否稳定获得超额收益。
- 同类排名和同类分位。
- 排名是否稳定，还是短期冲高。

风格和暴露：

- 资产配置比例。
- 行业、主题、地区和风格暴露。
- 是否与现有组合重复暴露。
- 是否存在风格漂移。

持有体验：

- 回撤是否超过用户可接受范围。
- 波动是否过大。
- 申赎规则和费率是否影响持有。
- 是否适合继续持有、降低关注或转入观察。

## Worker 流程

建议新增或扩展以下步骤：

1. 同步基金画像、基金经理、基金公司和基准信息。
2. 同步净值和规模等基础数据。
3. 计算风险收益快照。
4. 计算基准比较和同类比较。
5. 计算持仓暴露和风格稳定性。
6. 识别基金异常。
7. 将异常写入 `risk_event`。
8. 将重要异常接入 `review_task`。
9. 更新基金分析、日报和 AI 解释输入。

## API 和页面范围

API 第一版建议提供：

- 基金画像摘要。
- 风险收益摘要。
- 基准比较。
- 同类比较。
- 暴露和风格摘要。
- 基金异常列表。

页面第一版建议在基金分析页中增加深度模块：

- 基金画像。
- 基金经理和基金公司。
- 风险收益和持有体验。
- 基准和同类比较。
- 风格和暴露。
- 数据来源和数据日期。
- AI 基金解释。

## 复盘闭链接入

基金深度异常应进入复盘闭环：

- 写入 `risk_event`：记录基金经理变更、规模异常、回撤扩大、波动升高、跑输基准、同类排名下降等异常。
- 生成 `review_task`：只对重要异常生成事项，避免每日打卡压力。
- 写入日报：解释今天哪些基金需要关注。
- 进入 AI：解释基金画像、风险收益、基准比较和建议依据。
- 影响建议：只通过规则影响分级建议，AI 不直接生成建议。

## 验收标准

- FUND 不使用股票财报、公司估值或公司质量评分。
- ETF / LOF 深度分析不混入本设计。
- 基金评分能解释收益、回撤、波动、基准比较、同类比较和持有体验。
- 基金异常能生成风险事件或重要事项。
- 缺失、样本、估算和真实数据能被区分。
