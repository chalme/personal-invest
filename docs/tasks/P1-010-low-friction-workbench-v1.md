## Goal

把系统从“多个功能页面”升级成“低摩擦投资工作台”：用户打开首页就能知道今天有没有重要变化、为什么重要、是否需要介入，以及后续如何复盘。

## Scope

这一阶段先不做复杂任务系统，也不把用户变成每日打卡工具。重点是把风险、建议变化、组合变化和数据异常聚合成可读、可追踪、低打扰的重要事项。

## Concrete Changes

- 后端聚合 `risk_event`、`investment_advice`、`portfolio_snapshot`、`job_execution`、`market_trend_snapshot`，生成只读的重要事项列表。
- Dashboard 升级为今日概览，突出数据状态、市场状态、组合风险、重要事项和明日关注。
- 新增或改造建议 / 复盘入口，集中展示建议变化、来源规则、关键指标、置信度和后续表现。
- 强化周报 / 月报口径，基于组合快照展示最近 7 天和 30 天的变化。
- 清理少量体验债，把页面文案从“每日待办”改成“重要事项”与“低摩擦复盘”。

## Acceptance

- 30 秒内能看出今天是否需要介入。
- 没有重要变化时，明确显示“暂无需要立即处理事项”。
- 用户不处理建议时，系统仍能自动跟踪后续价格、净值、风险和建议变化。
- 周报和月报成为主要复盘窗口，而不是每日打卡。
