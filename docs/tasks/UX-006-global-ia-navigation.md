# UX-006: 全局信息架构与导航重组

## Status

- Status: `DONE`
- Priority: `P1`
- Owner: `Codex`

## Goal

将 Web UI 从“功能页面列表”重组为“投资工作流入口”，让用户打开系统后能按工作流理解：

```text
今日
观察
持仓
复盘
报告
设置
```

市场、行业、信号、策略、AI、回测不应全部平铺在主导航，而应作为二级入口或嵌入到对应工作流页面。

## Scope

- 调整主导航分组和命名。
- 让主导航表达核心流程：今天看什么、研究什么、持有什么、如何复盘、看什么报告、怎么配置。
- 将高级功能收纳到二级入口。
- 保留现有页面能力，减少入口噪音。
- 明确每个入口的首屏问题和用户意图。

## Suggested Navigation

| 主入口 | 包含内容 | 用户问题 |
|---|---|---|
| 今日 | Dashboard、数据状态、重要事项 | 今天是否需要介入 |
| 观察 | 观察池、股票研究、基金研究、ETF 研究 | 哪些资产值得继续看 |
| 持仓 | 组合风险、资产暴露、持仓建议 | 当前持仓最大风险是什么 |
| 复盘 | 重要事项、决策记录、后续结果 | 当时怎么判断，后来怎么样 |
| 报告 | 日报、月报、历史报告 | 今天和近期发生了什么 |
| 设置 | 数据源、任务、主题、密度、运行状态 | 系统是否正常、如何配置 |

## Out of Scope

- 不新增业务模型。
- 不改后端 API。
- 不做移动端导航专项。
- 不引入新路由框架。
- 不删除已有页面能力。

## Acceptance Criteria

- 主导航不再像功能清单。
- 用户 30 秒内能找到今日概览、观察资产、持仓风险、复盘记录和报告。
- 市场、行业、信号、策略、AI、回测作为二级入口时仍可发现。
- 导航文案保持专业、克制、低打扰。
