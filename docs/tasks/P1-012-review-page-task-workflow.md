- Completed At:

## Goal

让复盘页从实时聚合展示升级为持久化重要事项工作流，用户可以确认、延后或解决事项。

## Scope

只升级复盘页和相关 API 行为，不引入决策记录表单。Dashboard 仍只展示摘要，不承载完整处理流程。

## Concrete Changes

- `ReviewPage` 改为优先展示持久化 `review_task`。
- 支持确认、延后、解决三个轻量动作。
- 保留无事项空态：“暂无需要立即处理事项”。
- 任务失败、数据异常、建议变化和风险事项要有清晰标签、日期和来源。
- 类型定义补充 `ReviewTask` 和状态枚举。

## Acceptance

- 用户可以在复盘页更新事项状态。
- 延后事项在默认 OPEN 列表中隐藏，到期后重新出现。
- 已解决事项可在历史视图或筛选中查看。
- 无重要事项时不制造待办压力。

## Verification

- 前端构建通过。
- 有事项、无事项、任务失败、数据异常四类状态都有明确展示。
- 更新状态后列表和摘要刷新正确。

## Notes

- 本任务完成后，复盘页成为重要事项的主入口。
