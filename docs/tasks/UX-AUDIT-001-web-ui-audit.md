# UX-AUDIT-001: 当前 Web UI 审计

## Status

- Status: `TODO`
- Priority: `P1`
- Owner: `Codex`

## Goal

审计当前 Web UI，确认 Dashboard、股票页、持仓页、观察池、复盘页、日报和设置页为什么显得混乱，并输出可执行的改版原则。

目标不是直接改页面，而是先把问题讲清楚：

- 哪些导航入口干扰主流程。
- 哪些页面重点不突出。
- 哪些卡片、表格、Badge 和按钮不一致。
- 哪些数据可信度、空态、错误态和加载态表达不足。
- 哪些页面没有让用户快速判断是否需要介入。

## Design Read

本项目是桌面端个人投资工作台，长期自己使用，目标是专业、克制、清晰、高信息密度，接近金融工作台和 Linear-clean 产品 UI。

本任务可使用 `design-taste-frontend` skill 的审计、视觉层级、组件一致性和 pre-flight 思路，但不能套用 landing page、hero 大图、营销卡片或炫酷动效风格。

## Scope

- 审计 Dashboard、股票页、持仓页、观察池、复盘页、日报、设置页。
- 检查主导航是否像功能列表，而不是投资工作流。
- 检查首屏是否能回答页面最重要的问题。
- 检查卡片是否过多、表格是否过重、状态是否分散。
- 检查 `REAL` / `ESTIMATED` / `SAMPLE` / `MISSING` / `MIXED` 是否统一展示。
- 检查空态、错误态、加载态和任务失败文案是否完整。
- 检查桌面端信息密度、留白、层级、颜色和对比度。

## Out of Scope

- 不修改业务代码。
- 不重构页面。
- 不做移动端专项。
- 不引入新前端框架。
- 不做营销页式 redesign。
- 不做 hero 大图、夸张动效或装饰性视觉。

## Deliverables

- 新增或更新 `docs/ui-audit.md`。
- 每个关键页面至少包含：问题、影响、优先级、建议方向。
- 明确哪些问题进入 `UX-006`、`UX-007`、`UX-008`、`UX-009`。

## Acceptance Criteria

- 审计结论能解释当前 UI 为什么乱。
- 审计报告能指导后续任务，不只是主观审美意见。
- 改版原则符合桌面端金融工作台，而不是营销站。
- 低可信数据不能被包装成高置信结论。
