## Goal

规划股票页、持仓页、观察池和日报如何从“数据展示”升级为“结论优先”，并明确数据可信度在页面中的展示和降级口径。

## Principles

- 首屏先回答问题，再展示明细。
- 风险和需要用户复核的事项优先于普通机会。
- 页面必须区分事实、判断、风险和动作。
- `REAL` 数据可以支撑高置信结论；`ESTIMATED` 只能低置信解释；`SAMPLE` 只能演示；`MISSING` 只展示缺失。
- AI 只解释已有规则和数据，不生成黑盒高置信建议。

## Page Plans

### UX-002: 股票页研究结论化

首屏回答：

- 这家公司是否值得继续观察？
- 公司质量、估值、趋势和风险分别如何？
- 下一步应该看什么？

模块顺序：

1. 顶部研究结论卡：建议等级、一句话结论、置信度、数据日期。
2. 关键证据：趋势、估值、公司质量、行业、财报事件。
3. 风险点：财报异常、估值偏高、趋势转弱、行业转弱。
4. 下一步观察：等待回调、确认趋势、复核财报、降低关注。
5. 明细图表和指标表。

低可信降级：

- 财报为 `SAMPLE` 时，不展示高置信公司质量结论。
- 估值为 `ESTIMATED` 时，用“估算估值”文案，不能单独触发买入/卖出高优先级。
- 财报缺失时，保留价格和风险分析，但明确“基本面数据缺失”。

### UX-003: 持仓页组合决策化

首屏回答：

- 当前组合最大风险是什么？
- 哪些持仓需要优先处理？
- 股票、ETF、基金暴露是否集中？

模块顺序：

1. 组合风险结论：整体风险、最大单一风险、是否需要介入。
2. 优先复核资产：减仓关注、卖出关注、仓位过重、回撤扩大。
3. 暴露结构：资产类型、行业/主题、单一资产集中度。
4. 盈亏贡献：收益/亏损来源和风险对应关系。
5. 持仓明细表。

低可信降级：

- 价格/净值不是 `REAL` 时，盈亏和仓位提示标记为低置信。
- ETF 暴露为 `ESTIMATED` 时，只提示主题暴露，不做精确集中度结论。
- 基金画像缺失时，不做基金类型集中判断。

### UX-004: 观察池研究状态化

首屏回答：

- 我正在研究什么？
- 哪些资产值得继续观察？
- 哪些关注已经失效？

模块顺序：

1. 观察池摘要：高优先级、风险升高、数据缺失、长期未更新。
2. 研究状态列表：关注理由、建议等级、最新变化、下一步观察。
3. 资产类型筛选：股票、ETF、FUND。
4. 清理入口：降低优先级、暂停关注、移除。

低可信降级：

- 数据缺失资产进入“数据待补齐”分组。
- 样本数据资产不进入高优先级机会排序。
- 估算暴露只作为辅助标签。

### UX-005: 日报投资简报化

首屏回答：

- 今天最重要的变化是什么？
- 有没有需要人工复核的风险？
- 明天最应该关注什么？

模块顺序：

1. 今日结论：市场状态、组合状态、是否需要介入。
2. 重要事项：财报异常、ETF 深度异常、基金深度异常、持仓风险。
3. 市场和行业：热门、冷门、轮动、防守、过热。
4. 资产分区：股票、ETF、FUND 分开解释。
5. 明日观察清单。
6. Markdown 原文归档。

低可信降级：

- 如果市场数据为 `SAMPLE`，日报顶部提示“演示数据”。
- 如果某模块为 `MISSING`，日报该模块只显示缺失，不生成判断。
- 如果整体为 `MIXED`，日报结论必须提示哪些模块低可信。
