# UX-008: Dashboard 今日工作台升级

## Status

- Status: `TODO`
- Priority: `P1`
- Owner: `Codex`

## Goal

将 Dashboard 从“模块总览”升级为“今日工作台”，首屏只回答最关键的问题：

```text
数据是否新鲜？
今天市场如何？
组合有没有风险？
哪些事项需要看？
明天关注什么？
```

## Scope

- 重排 Dashboard 首屏信息结构。
- 突出数据新鲜度和数据可信度。
- 突出市场状态和组合风险。
- 突出重要事项和建议变化。
- 增加清晰的无重要事项空态。
- 保留下钻入口，不在首页堆满所有模块。

## Required Sections

- 数据状态：最新数据日期、预计交易日、新鲜度、可信度。
- 今日结论：市场状态、组合风险、重要事项数量。
- 需要关注：高风险、建议变化、数据异常、任务失败。
- 明日关注：需要复核的资产、行业或数据缺口。
- 下钻入口：观察、持仓、复盘、报告。

## Out of Scope

- 不新增数据库表。
- 不让 AI 自由生成买卖建议。
- 不接新闻宏观源。
- 不做复杂收益归因。
- 不做移动端专项。

## Acceptance Criteria

- 用户打开 Dashboard 后 30 秒内能判断今天是否需要介入。
- 没有重要事项时显示“暂无需要立即处理事项”，不制造焦虑。
- 数据日期和可信度始终可见。
- 首页不再像所有功能模块的堆叠。
- 重要事项能跳转到复盘页，持仓风险能跳转到持仓页。
