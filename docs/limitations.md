# 系统限制

本文集中说明系统边界。它不是免责声明的替代品，而是工程和产品设计时必须遵守的约束。

## 结论先行

Personal Invest 是个人投资研究和复盘工具，不是交易系统。它的价值是把数据、规则、风险和复盘组织起来，帮助用户做更有纪律的判断。

它不能替代：

- 专业投资顾问。
- 券商交易系统。
- 付费数据终端。
- 多用户资产管理平台。
- 投资收益保证。

## 投资决策限制

| 限制 | 说明 |
|---|---|
| 不自动下单 | 系统不会连接券商，也不会执行买卖指令 |
| 不承诺收益 | 所有信号只是规则结果和风险提示 |
| 不替代人工判断 | 缺数据、过期数据、异常行情都需要人工复核 |
| 不做高频交易 | 系统以日频研究、日报和复盘为主 |
| 不做合规投顾 | 不提供个性化持牌投顾服务 |

## 数据限制

| 限制 | 影响 | 处理方式 |
|---|---|---|
| 免费公开源不稳定 | 可能超时、断连、字段变化 | 多真实源 fallback、探针、缓存、缺失态 |
| 交易日历第一版按工作日估算 | 法定节假日可能误判新鲜度 | 页面提示 `ESTIMATED`，后续可接真实交易日历 |
| 部分财务 / 基金 / ETF 深度源不完整 | 页面可能显示 `MISSING` | 不造数据，明确提示缺失 |
| 真实历史缓存不是最新数据 | 不能驱动高置信建议 | 标记 `REAL_CACHED` / `STALE` |
| provider 字段不一致 | 某些字段可能为空 | 用 `missing_fields` 暴露，不伪造 |

## real-only 限制

系统禁止用非真实数据填充运行时结果。

禁止：

- `sample`。
- `mock`。
- `demo`。
- `estimated`。
- `deterministic_estimate`。
- `built_in_sample`。

允许：

- 当前真实源数据。
- 真实历史缓存。
- 明确的缺失态。

代价：页面可能不完整，但这比伪造数据更安全。

## 架构限制

| 限制 | 原因 |
|---|---|
| 单机优先 | 当前以个人本地/个人服务器为目标 |
| SQLite 并发有限 | 适合个人使用，不适合多人高并发写入 |
| Parquet 本地文件管理 | 分析方便，但需要谨慎处理并发写和备份 |
| 无应用内多用户权限 | 访问保护依赖 Cloudflare Access / 反向代理 |
| 无复杂任务队列 | 当前 worker 更偏脚本化每日任务 |
| 无强 SLA | 免费数据源和个人服务器无法提供商业级可用性 |

## AI 分析限制

AI 页面只解释系统已有数据和规则。

不能：

- 绕过风控给出下单指令。
- 在缺数据时编造结论。
- 替代真实数据源。
- 保证预测准确。

应该：

- 引用已有指标和规则。
- 说明数据日期和来源。
- 说明风险边界。
- 在数据缺失时降级为“无法判断”。

## 回测限制

当前回测是基础观察池验证，不是专业量化交易系统。

限制包括：

- 不保证无未来函数之外的所有真实交易成本建模。
- 不完整模拟滑点、冲击成本、税费和交易限制。
- 不支持复杂组合优化和多策略资金调度。
- 不代表未来收益。

## 安全限制

生产环境建议：

- 使用 Cloudflare Access 或等价访问控制保护前端和 API。
- 不让后端源站端口绕过访问保护暴露公网。
- 定期备份 `storage/`、`data/` 和 `reports/`。
- 不在仓库提交真实 token、账户、私钥或敏感配置。

当前不提供：

- 应用内用户注册。
- 角色权限管理。
- 审计日志体系。
- 多租户隔离。

## 运维限制

| 场景 | 当前能力 | 限制 |
|---|---|---|
| 本地开发 | `make dev` | 依赖本机 uv/node/corepack |
| 生产运行 | `make prod-server` / systemd | 需要人工配置服务和域名 |
| 数据同步 | `make daily` | 外部源失败会降级为缓存或缺失 |
| 数据恢复 | `make backup` | 恢复 runbook 仍需持续补齐 |
| 健康检查 | `make health` / 探针脚本 | 不等同于完整监控系统 |

## 何时不应该使用系统结论

以下情况只应记录和观察，不应按系统建议直接行动：

1. 核心数据集为 `MISSING`。
2. 关键行情为 `REAL_CACHED` 或 `STALE`。
3. 数据源探针显示主要 provider 大面积失败。
4. 观察资产缺少真实财务、净值或 ETF 深度数据。
5. 市场处于重大事件、停牌、临时休市或极端波动环境。
6. 用户没有记录成本、仓位或持仓信息。
