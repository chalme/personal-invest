# UI / UX 设计要求

## 设计目标

页面不能是粗糙后台管理系统，而应该是现代金融仪表盘。

关键词：

- 极简
- 清晰
- 信息密度高
- 风险突出
- 图表友好
- 明暗主题
- 移动端可浏览

## 前端升级原则

下一阶段前端要从“多页面数据展示”升级为“今日概览 + 重要变化提醒”。用户进入系统后，应能在 30 秒内知道今天是否有重要风险、建议变化或需要介入的事项。

核心原则：

- 风险优先：高风险、卖出关注、减仓关注、数据异常必须优先展示。
- 低打扰优先：页面不制造每日待办，只在重要变化、风险升高或真实决策场景中提醒用户介入。
- 建议可操作：建议不能只显示标签，必须给出原因、证据、风险和可选下一步动作。
- 数据日期可见：所有市场、行业、资产、建议、日报、AI 解释都要展示数据日期或更新时间。
- 状态完整：加载、空状态、错误、任务运行中、任务失败都要有清晰文案和下一步动作。
- 移动端可浏览：移动端至少能浏览今日风险、建议、组合状态和日报摘要。
- 专业密度：保持金融工作台的信息密度，不做营销页式大图、夸张 hero 或装饰性动画。

执行 UI 升级时，可以使用现有 frontend/design skill 做审计和预检，重点检查导航、信息层级、表格、状态、响应式和视觉一致性。

## 硬性体验标准

| 维度 | 要求 |
|---|---|
| 首屏 | 1 秒内显示骨架屏或核心卡片 |
| 主题 | 支持明亮 / 暗色主题 |
| 图表 | 使用 ECharts |
| 表格 | 支持排序、筛选、分页 |
| 数据时间 | 所有页面显示数据日期和更新时间 |
| 任务状态 | 有进度、成功、失败原因 |
| 风险事件 | 高优先级风险突出展示 |
| 分级建议 | 明确展示建议等级、一句话结论、触发原因、关键指标、风险、复核动作、置信度和数据日期 |
| 空状态 | 没有重要变化时显示“暂无需要立即处理事项”，并给出可选浏览入口 |
| 危险操作 | 删除、重跑、覆盖报告二次确认 |

## Dashboard 必须回答

1. 今天数据是不是最新？
2. 当前市场状态是什么？
3. 热门、冷门、轮动、防守、过热行业分别是什么？
4. 我的持仓有什么风险？
5. 我的组合暴露是否过度集中？
6. 今天哪些资产出现重要建议变化？
7. 没有重要变化时，是否明确显示暂无需要立即处理事项？
8. 明天我应该重点看什么？

## 工作流导航方向

后续导航可以从“页面列表”逐步升级为“投资工作流”：

- 今日：Dashboard 和每日更新状态，回答今天是否有重要变化。
- 观察：观察池、个股分析、基金分析，回答哪些资产值得继续研究。
- 持仓：组合风险、资产暴露、盈亏、集中度和持仓建议。
- 建议：买入关注、持有、减仓关注、卖出关注和复核动作。
- 复盘：日报、月报、决策记录和建议结果跟踪。
- 宏观：市场环境、行业/主题、宏观指标和新闻事件影响。
- 设置：数据源、策略阈值、AI、主题和密度。

## 关键页面口径

观察池回答“我正在研究什么”，不等于已经买入。

个人持仓回答“我真实持有什么”，重点展示盈亏、仓位、组合暴露、集中度、风险和分级建议。

个股分析只分析股票。

基金分析覆盖 ETF 和场外基金，但 ETF 使用价格型口径，场外基金使用净值型口径。

AI 分析解释规则生成的建议，不直接绕过规则给出黑盒结论。

## 关键页面升级方向

Dashboard 不再只是总览，而是今日概览与重要变化提醒，优先展示数据可信度、市场状态、组合风险、关键建议变化和明日关注。没有重要变化时，应明确显示“暂无需要立即处理事项”。

观察池要展示资产当前状态、建议等级、关注理由、最新风险、数据日期和下一步观察点。

持仓页要从表格中心升级为组合风险页，强化资产暴露、集中度、盈亏、建议状态和复核动作。

AI 页要按时间窗口组织：今日解释、明日关注、近 1 周复盘、近 1 月总结、宏观新闻影响。

报告页要从 Markdown 阅读器升级为日报、月报和复盘摘要入口。

## 今日建议展示口径

今日建议不能只显示标签。每条建议至少展示：

- 一句话结论
- 建议等级
- 触发原因
- 关键证据
- 风险提醒
- 复核动作
- 数据日期

示例结构：

`买入关注`：行业升温、趋势改善、成交配合，但估值偏高，明天优先复核回调或确认信号。

## 市场与行业展示口径

Dashboard 展示行业摘要，行业页展示完整分析。

行业页至少包含：

- 热门行业
- 冷门行业
- 轮动行业
- 防守行业
- 过热行业
- 行业强弱完整排名
- 行业对应的股票、ETF、基金观察对象
