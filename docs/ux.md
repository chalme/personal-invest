# UI / UX 设计要求

## 设计目标

页面不能是粗糙后台管理系统，而应该是现代金融仪表盘。

关键词：

- 极简
- 清晰
- 信息密度高
- 风险突出
- 图表友好
- 明暗主题
- 移动端可浏览

## 页面列表

- Dashboard 首页
- 市场趋势页
- 行业强弱页
- 个股分析页
- 自选股页
- 持仓页
- 策略信号页
- 回测页
- 日报页
- AI 分析页
- 设置页

## 硬性体验标准

| 维度 | 要求 |
|---|---|
| 首屏 | 1 秒内显示骨架屏或核心卡片 |
| 主题 | 支持明亮 / 暗色主题 |
| 图表 | 使用 ECharts |
| 表格 | 支持排序、筛选、分页 |
| 数据时间 | 所有页面显示数据日期和更新时间 |
| 任务状态 | 有进度、成功、失败原因 |
| 风险事件 | 高优先级风险突出展示 |
| 空状态 | 给明确下一步操作 |
| 危险操作 | 删除、重跑、覆盖报告二次确认 |

## Dashboard 必须回答

1. 今天数据是不是最新？
2. 当前市场状态是什么？
3. 我的持仓有什么风险？
4. 今天出现了哪些信号？
5. 明天我应该重点看什么？
