# UI / UX 设计要求

## 设计目标

页面不能是粗糙后台管理系统，而应该是现代金融仪表盘。

关键词：

- 极简
- 清晰
- 信息密度高
- 风险突出
- 图表友好
- 明暗主题
- 移动端可浏览

## 页面列表

- Dashboard 首页
- 市场趋势页
- 行业强弱页
- 观察池页
- 个股分析页
- 基金分析页
- 个人持仓页
- 策略信号页
- 回测页
- 日报页
- AI 分析页
- 设置页

## 硬性体验标准

| 维度 | 要求 |
|---|---|
| 首屏 | 1 秒内显示骨架屏或核心卡片 |
| 主题 | 支持明亮 / 暗色主题 |
| 图表 | 使用 ECharts |
| 表格 | 支持排序、筛选、分页 |
| 数据时间 | 所有页面显示数据日期和更新时间 |
| 任务状态 | 有进度、成功、失败原因 |
| 风险事件 | 高优先级风险突出展示 |
| 分级建议 | 明确展示建议等级、一句话结论、触发原因、关键指标、风险、复核动作、置信度和数据日期 |
| 空状态 | 给明确下一步操作 |
| 危险操作 | 删除、重跑、覆盖报告二次确认 |

## Dashboard 必须回答

1. 今天数据是不是最新？
2. 当前市场状态是什么？
3. 热门、冷门、轮动、防守、过热行业分别是什么？
4. 我的持仓有什么风险？
5. 我的组合暴露是否过度集中？
6. 今天哪些资产有具象化分级建议？
7. 明天我应该重点看什么？

## 关键页面口径

观察池回答“我正在研究什么”，不等于已经买入。

个人持仓回答“我真实持有什么”，重点展示盈亏、仓位、组合暴露、集中度、风险和分级建议。

个股分析只分析股票。

基金分析覆盖 ETF 和场外基金，但 ETF 使用价格型口径，场外基金使用净值型口径。

AI 分析解释规则生成的建议，不直接绕过规则给出黑盒结论。

## 今日建议展示口径

今日建议不能只显示标签。每条建议至少展示：

- 一句话结论
- 建议等级
- 触发原因
- 关键证据
- 风险提醒
- 复核动作
- 数据日期

示例结构：

`买入关注`：行业升温、趋势改善、成交配合，但估值偏高，明天优先复核回调或确认信号。

## 市场与行业展示口径

Dashboard 展示行业摘要，行业页展示完整分析。

行业页至少包含：

- 热门行业
- 冷门行业
- 轮动行业
- 防守行业
- 过热行业
- 行业强弱完整排名
- 行业对应的股票、ETF、基金观察对象
