# Product Backlog

本文档记录个人投资系统的产品打磨计划，只包含产品体验、业务功能和投资分析能力相关事项。

运维类事项记录在 docs/operation-backlog.md。

## 当前产品定位

本系统定位为：浏览器可访问、页面优雅、体验良好的个人投资研究、持仓管理、组合复盘和分级买卖建议系统。

核心目标不是自动交易，而是辅助完成市场趋势判断、行业强弱判断、股票分析、公司财报分析、基金分析、观察池跟踪、个人持仓管理、组合暴露分析、分级买卖建议、策略信号生成、回测验证、投资日报和 AI 辅助解释。

业务规划详见 `docs/business-roadmap.md`。


## 已完成能力快照

截至当前版本，以下能力已经完成第一版，不应再作为当前未完成主线重复拆解：

- 模型收敛：轻量迁移体系、`instrument` 资产主数据、ETF 独立口径、建议规则追溯、行业/主题映射、组合历史快照。
- 多资产闭环：股票、ETF、场外基金均可进入观察池、持仓、信号、建议、日报、AI 与复盘链路。
- 股票财报 V1：财报/指标/估值/公司质量快照、财报事件、股票页展示、AI 与重要事项接入。
- 场外基金深度 V1：基金画像、风险收益、基准/同类比较、暴露、深度事件、基金页、AI 与复盘接入。
- ETF 深度 V1：ETF 画像、暴露、流动性、风险收益、跟踪质量、页面、AI、日报与复盘闭环。
- 低摩擦复盘闭环：`review_task`、`decision_record`、`decision_outcome`、ReviewPage、Dashboard/Portfolio 摘要入口。
- 桌面端体验：亮/暗主题、标准/紧凑密度和设置持久化；移动端响应式仍后置。

## P1：股票基本面与财报分析能力

目标：让股票分析里的基本面、估值和公司质量评分有真实财报与估值数据支撑，而不是停留在静态评分或占位指标。

该能力只适用于 `STOCK`。ETF、LOF 和场外基金不使用公司财报、公司估值或公司质量评分。

股票财报分析和基金深度分析是并列路线：股票看公司，基金看产品、经理、策略和风险收益，ETF 看指数、主题、流动性和跟踪质量。

待办：

- [ ] 新增财报数据层：利润表、资产负债表、现金流量表、分红和回购
- [ ] 新增财务指标快照：营收增速、净利润增速、毛利率、净利率、ROE、ROA、负债率、经营现金流和自由现金流
- [ ] 新增估值快照：PE、PB、PS、股息率、历史分位、行业分位和估值区间
- [ ] 新增财报事件：业绩预告、定期报告、利润大幅变化、现金流异常、资产减值、审计意见和管理层变化
- [ ] 将财报和估值结果反哺 `fundamental_score`、`valuation_score` 和 `risk_score`
- [ ] 财报异常进入 `risk_event` 和重要事项池
- [ ] AI 支持解释财报变化、估值变化和公司质量评分来源

验收标准：

- [ ] 股票基本面评分可追溯到财务指标和数据日期
- [ ] 估值判断能说明当前估值相对历史和行业的位置

## P1：场外基金深度分析能力

目标：让基金分析从“净值、收益、回撤、波动”的 MVP 口径，升级为基金产品、基金经理、风险收益、同类比较、基准比较和组合适配分析。

基金深度分析不使用股票财报、公司估值或公司质量评分。基金的“基本面”是基金产品、经理、策略、持仓、风格、风险收益和同类比较。

该路线与股票财报分析并列推进，不把基金深度分析做成股票模型的子集。

ETF 已有 `ETF_PRICE` 基础分析，但 ETF 深度分析单独排期，不混入场外基金深度分析。

第一阶段基金深度范围：

- 场外基金：基金画像、基金经理、基金公司、风险等级、基金类型、基准、费率、申赎规则、阶段收益、风险收益和同类比较。

待办：

- [ ] 新增基金画像：基金类型、风险等级、基金经理、基金公司、成立时间、规模、基准、费率和申赎规则
- [ ] 新增风险收益指标：夏普、卡玛、下行波动、最大回撤修复时间、近 1/3/6/12 月收益
- [ ] 新增基准和同类比较：相对基准收益、同类排名、同类分位、是否稳定获得超额
- [ ] 新增基金风格和暴露：股票/债券/货币/商品/海外比例，行业、主题、地区和风格暴露
- [ ] 将基金深度结果接入基金分析页、重要事项池、分级建议、日报和 AI 解释

验收标准：

- [ ] 基金评分能解释收益、回撤、波动、基准比较、同类比较和持有体验
- [ ] 场外基金能解释基金经理、基金类型、风险等级、费率和风格稳定性
- [ ] 基金异常能生成风险事件或重要事项

目标：让 AI 分析成为规则建议解释器，而不是绕过规则的黑盒交易判断。

待办：

- [ ] 增加外部 LLM 开关
- [ ] 增加模型配置
- [ ] 增加 AI 结果缓存
- [ ] 增加引用数据来源
- [ ] 增加事实、推断、风险分区
- [ ] 增加防幻觉约束
- [ ] AI 解释规则生成的分级建议
- [ ] AI 输出建议时必须附规则依据、关键指标、数据日期和风险说明

验收标准：

- [ ] AI 回答必须基于系统已有数据
- [ ] AI 输出必须包含不确定性和风险边界
- [ ] AI 不直接凭空生成建议
- [ ] AI 不替代人工最终决策
