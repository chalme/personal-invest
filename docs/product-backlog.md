# Product Backlog

本文档记录个人投资系统的产品打磨计划，只包含产品体验、业务功能和投资分析能力相关事项。

运维类事项记录在 docs/operation-backlog.md。

## 当前产品定位

本系统定位为：浏览器可访问、页面优雅、体验良好的个人投资研究与复盘系统。

核心目标不是自动交易，而是辅助完成市场趋势判断、行业强弱判断、股票分析、基金分析、观察池跟踪、持仓风险分析、策略信号生成、回测验证、投资日报和 AI 辅助解释。

## 任务分层

本文档只记录长期产品需求池。已经确认要执行的任务进入 `docs/task-board.md`；复杂任务再拆到 `docs/tasks/*.md`。

当前执行主线：

- 可用性
- 数据可信度
- 股票 + 基金双资产分析

## P1：个股分析页打磨

目标：把个股页从指标展示页升级为公司研究结论页。

打开个股页后，应优先回答：

- 这家公司现在是否值得继续观察？
- 当前趋势是否改善？
- 基本面质量怎么样？
- 当前估值是否偏贵？
- 主要风险在哪里？
- 下一步应该观察什么？

待办：

- [ ] 增加顶部结论卡片
- [ ] 增加综合状态说明
- [ ] 增加趋势、基本面、估值、资金、行业、风险的分项解释
- [ ] 增加风险点区域
- [ ] 增加下一步观察点区域
- [ ] 优化价格趋势图
- [ ] 优化评分雷达图说明
- [ ] 后续接入财务趋势图
- [ ] 后续接入估值历史分位

验收标准：

- [ ] 不需要读完整表格，也能获得明确结论
- [ ] 页面能区分事实、判断和风险
- [ ] 页面不直接给买卖指令

## P1：基金分析能力

目标：让基金和股票一样进入研究、观察、持仓、信号和复盘闭环，但不把基金硬套进股票基本面/估值口径。

第一版基金范围：

- 场外公募基金：以净值、收益、回撤、波动、基金类型和风险标签为核心。
- 场内 ETF / LOF：兼容现有行情日线管线，可按 ETF 资产类型展示。

待办：

- [ ] 引入资产类型：股票、ETF、基金
- [ ] 新增基金净值数据集
- [ ] 新增基金分析快照
- [ ] 新增基金分析页
- [ ] 观察池支持股票和基金
- [ ] 持仓页支持基金份额、成本净值、当前净值
- [ ] 信号、日报、AI 分析支持基金解释

验收标准：

- [ ] 股票和基金能在观察池中区分
- [ ] 基金分析不使用股票基本面/估值默认分
- [ ] 基金持仓能进入组合风险总览
