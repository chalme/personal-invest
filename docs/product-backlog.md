# Product Backlog

本文档记录个人投资系统的产品打磨计划，只包含产品体验、业务功能和投资分析能力相关事项。

运维类事项记录在 docs/operation-backlog.md。

## 当前产品定位

本系统定位为：浏览器可访问、页面优雅、体验良好的个人投资研究、持仓管理、组合复盘和分级买卖建议系统。

核心目标不是自动交易，而是辅助完成市场趋势判断、行业强弱判断、股票分析、基金分析、观察池跟踪、个人持仓管理、组合暴露分析、分级买卖建议、策略信号生成、回测验证、投资日报和 AI 辅助解释。

业务规划详见 `docs/business-roadmap.md`。

## 任务分层

本文档只记录长期产品需求池。已经确认要执行的任务进入 `docs/task-board.md`；复杂任务再拆到 `docs/tasks/*.md`。

当前执行主线：

- 可用性
- 数据可信度
- 股票 + ETF + 场外基金三类资产
- 个人持仓与组合暴露
- 规则生成分级买卖建议，AI 负责解释

## P0：资产类型、个人持仓与分级建议底座

目标：先把业务底座讲清楚并落到系统模型里，让观察池、个人持仓、组合复盘、信号、日报和 AI 都能围绕同一套资产类型和建议等级工作。

第一阶段资产类型：

- 股票：`STOCK`
- 场内 ETF / LOF：`ETF`
- 场外公募基金：`FUND`

第一阶段建议等级：

- 继续观察
- 买入关注
- 持有
- 减仓关注
- 卖出关注

待办：

- [ ] 引入资产类型：股票、ETF、场外基金
- [ ] 明确观察池和个人持仓的区别
- [ ] 个人持仓第一阶段采用单组合模型
- [ ] 观察池、持仓、信号、风险事件返回资产类型
- [ ] 新增分级建议枚举
- [ ] 建议结果包含触发原因、关键指标、风险说明、置信度、数据日期
- [ ] 规则负责生成建议，AI 只负责解释建议

验收标准：

- [ ] 股票、ETF、基金能在观察池和持仓中区分
- [ ] 观察池不等于持仓
- [ ] 持仓页能表达组合暴露、盈亏、集中度、风险和建议
- [ ] 每条建议可追溯到规则、指标、数据日期和风险说明

## P1：个股分析页打磨

- 当前趋势是否改善？
- 基本面质量怎么样？
- 当前估值是否偏贵？
- 主要风险在哪里？
- 当前分级建议是什么？
- 下一步应该观察什么？

- [ ] 增加顶部结论卡片
- [ ] 增加综合状态说明
- [ ] 增加趋势、基本面、估值、资金、行业、风险的分项解释
- [ ] 增加风险点区域
- [ ] 增加分级建议区域
- [ ] 增加下一步观察点区域
- [ ] 优化价格趋势图
- [ ] 优化评分雷达图说明
- [ ] 后续接入财务趋势图
- [ ] 后续接入估值历史分位

验收标准：

- [ ] 不需要读完整表格，也能获得明确结论
- [ ] 页面能区分事实、判断和风险
- [ ] 页面展示规则生成的分级建议和依据

## P1：基金分析能力

目标：让基金和股票一样进入研究、观察、持仓、信号和复盘闭环，但不把基金硬套进股票基本面/估值口径。

第一版基金范围：

- 场外公募基金：以净值、收益、回撤、波动、基金类型和风险标签为核心。
- 场内 ETF / LOF：兼容现有行情日线管线，可按 ETF 资产类型展示。

待办：

- [ ] 新增基金净值数据集
- [ ] 新增基金分析快照
- [ ] 新增基金分析页
- [ ] 观察池支持股票和基金
- [ ] 持仓页支持基金份额、成本净值、当前净值
- [ ] 信号、日报、AI 分析支持基金解释和分级建议

验收标准：

- [ ] 股票和基金能在观察池中区分
- [ ] 基金分析不使用股票基本面/估值默认分
- [ ] 基金持仓能进入组合风险总览
- [ ] ETF 和场外基金使用不同建议口径
